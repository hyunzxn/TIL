# text, utext

타임리프의 기능 중 text 와 utext에 대해 정리하고자 합니다.



## text

컨트롤러에서 모델에 담긴 문자 데이터를 타임리프를 이용하여 화면에 출력할 때 사용하는 속성이다. 

<br>

**사용방법**

`th:text="${data}"`

ex) `<li>th:text 사용 <span th:text="${data}"></span></li>` → 태그의 attribute에 넣어줌

- th:text  속성 사용
- ${} 담을 데이터를 { } 안에 넣어주면 된다. 이 때 데이터 값 자체를 넣는 것이 아니라 attributeName을 넣어줘야한다.

<br>

만약 모델에 담긴 데이터를 HTML 태그의 attribute가 아니라 내용에서 바로 사용하고 싶으면 다음과 같이 써주면 된다.

`[[${data}]]`

ex) `<li>컨텐츠 안에서 직접 출력하기 = [[${data}]]</li>`  → 태그 내용에서 바로 사용

<br>

## utext

타임리프는 th:text 를 사용하면 자체적으로 **이스케이프** 처리를 해준다. 이를 다시 언이스케이프 해주기 위해서 사용하는 속성이다. 

<br>

**Escape(이스케이프)**

HTML은 < , > 와 같은 특수문자를 기반으로 작성된다. 그래서 이런 특수문자를 구분할 수 있는 별도의 처리를 해줘야한다. 예를 들어서 `<b>Spring</b>`을 이용해서 볼드체로 진하게 표시하려고 HTML에서 `<b>Spring</b>`을 사용하면 어떻게 될까?

<br>

타임리프의 th:text 를 사용하면 기본적으로 이스케이프 처리를 해주기 때문에 다음과 같이 표시가 된다. `&lt;b&gt;Spring!&lt;/b&gt` 이스케이프 처리가 되어서 < , > 와 같은 기호가 &lt, b&gt 같은 기호로 바뀌어지는 것을 확인할 수 있었다. 이런 기호를 HTML 엔티티라고 한다. 

<br>

이스케이프 처리를 제대로 해주지 않으면 개발자의 의도대로 되지 않을 경우가 많이 생기기 때문에 이스케이프 처리는 중요하다. 

<br>

그렇지만 가끔은 이스케이프 처리를 원하지 않고 정말 HTML 태그를 그대로 사용하고 싶어질 수도 있다. 그럴 때를 대비해서 타임리프에서 제공하는 기능이 th: utext 이다.

<br>

**사용방법**

`<li>th:utext = <span th:utext="${data}"></span></li>`

- th:utext 속성을 사용해준다. 그 외에 데이터를 넣는 방법은 th:text 와 동일하다.

<br>

만약 모델에 담긴 데이터를 HTML 태그의 attribute가 아니라 내용에서 바로 사용하고 싶으면 다음과 같이 써주면 된다.

`[(${data})]`
